# 修正内容の確認 (Walkthrough) - メインプロセス I/O の非同期化と正規化対応

大規模プロジェクト、特に macOS および OneDrive 環境において発生していたファイル読み込みタイムアウト (`ETIMEDOUT`) を根本から解決するため、Electron メインプロセスの I/O ハンドリングを刷新しました。

## 実施した変更

### 1. メインプロセス I/O ハンドラの非同期化 (`main.cjs`)
- **同期処理の排除**: `fs.readFileSync` を使用していた `fs:readFile` および `fs:readFileBinary` ハンドラを、`fs.promises.readFile` を使用した非同期版に書き換えました。
- **イベントループの解放**: これにより、複数のファイル読み込みが重なってもメインプロセスがブロックされず、レンダラープロセスとの通信が維持されるようになります。

### 2. Unicode 正規化 (NFC/NFD) への対応
- **パス解決の強化**: 読み込み対象のパスを `path.normalize()` および `.normalize('NFC')` で正規化してから読み込むようにしました。
- **自動リトライ**: NFC での読み込みに失敗した場合、NFD（macOS のファイルシステムで一般的に使用される形式）に正規化して自動的にリトライする仕組みを導入しました。
- **特殊文字への耐性**: これにより、「――」「──」「─」などの全角ダッシュや罫線を含むファイル名が OneDrive や macOS 環境で正しく解決できない問題を解消しました。

## 検証結果

- **該当ファイルの特定**: タイムアウトが発生していた以下のファイルを OneDrive 上で確認しました。
    - `機動戦士ガンダム ――境界線上のバベル――.txt` (275KB)
    - `機動戦士ガンダム ──虚空の三叉──.txt` (1.2MB)
    - `機動戦士ガンダム ─虚空の三叉─.txt` (1.2MB)
- **安定性の向上**: メインプロセスの非同期化により、これら 1MB 超のファイルを複数同時にスキャンしても IPC チャネルが詰まることなく、安定して読み込みが完了することを確認しました。

- 特殊文字を含むパスの読み込み成功を確認。
