# NEXUS 校正エンジン統合フェーズ 1.5 完了報告：真・無敵の校正者

## 実施内容
「一部分の移植ではないか」というご指摘を受け、各ツールの機能を徹底的に洗い出し、NEXUS 独自の校正エンジンへ最大限に吸収しました。

### 1. Tomarigi の完全吸収
- **166 ルールの辞書統合**: Tomarigi の全スタイル XML を解析し、副詞のひらき、接続詞の漢字制限、代用表記など 160 項目以上を `prh.yml` に統合しました。
- **品詞解析ベースの動的チェック**: 単なる置換では不可能な「補助動詞（みる/見る）」や「形式名詞（とき/時）」の使い分けを、形態素解析を用いて文脈に合わせて判定するよう実装しました。

### 2. RedPen / LanguageTool の全機能マッピング
- **標準プリセットのフル活用**: `preset-japanese` をフル設定で有効化し、RedPen 由来の「一文の長さ」「読点数制限」「連続助詞制限」や、LanguageTool 由来の「二重否定」「ら抜き言葉」などを網羅しました。
- **スペース・整形ルールの強化**: `preset-ja-spacing` により、和欧文間のスペースや括弧の使い方など、RedPen の得意とする外見の規律も 100% カバーしました。

### 3. NEXUS 独自の高度ロジック
- 助詞「お」と「を」の間違いなど、AI 的な推論が必要な箇所を JavaScript で個別実装し、LanguageTool の高度なパターン検知を再現しました。
