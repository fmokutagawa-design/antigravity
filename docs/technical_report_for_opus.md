# テクニカル・レポート：NEXUS 校正・監査エンジン統合プロジェクト

## 0. プロジェクト概要
NEXUS エディタにおいて、物語の整合性（設定矛盾）を検知する「物語整合性エンジン」の実装、および 1,300 ルールを超える「最強の言語ハブ（textlint）」の UI 統合を完了しました。

## 1. 改善指示書 8本セット（実装済み）
`main` ブランチにおいて、以下の 8 つの主要なロジック強化を完了しています。

| 指示書 | 内容 | 主な変更点 |
| :--- | :--- | :--- |
| **1/8** | 会話文分離 | 「」内を校正対象から除外・分離するロジックの実装 |
| **2/8** | パス管理改善 | `config_loader.py` によるハードコードパスの解消 |
| **3/8** | 抽出制限解除 | ナレッジ抽出時の 100 行制限を撤廃 |
| **4/8** | ホワイトリスト | 作品固有用語を誤検知させない辞書機能 |
| **5/8** | フィルタリング UI | カテゴリ・ファイル別の校正結果抽出表示 |
| **6/8** | 監査の非同期化 | `BackgroundTasks` による進捗表示と非ブロッキング処理 |
| **7/8** | エイリアスリスト | 表記の揺れ（山田／山田隆志等）の名寄せ監査 |
| **8/8** | 時系列・生存監査 | 物語の進行に伴うキャラ状態（生存等）の整合性チェック |

## 2. 物語整合性エンジン (Phase 2)
設定資料を「マスター（正解）」とし、本文との矛盾を検知する Layer 2 監査。
- **作品の自動隔離**: `project_utils.py` により、ファイルパスからプロジェクトを特定。
- **詳細属性照合**: キャラクターの「瞳の色」「出身」「武器」等の設定矛盾を自動検知。
- **対話型ナレッジ管理**: `KnowledgeSuggestionBanner` による AI 提案と YAML 自動注入。

## 3. 言語ハブ統合 (Phase 1)
- **最強校正 UI**: `textlint` (1,300+ rules) の指摘を `AuditReportWindow` に統合。
- **ジャンプ機能**: 指摘箇所への正確なスクロールと選択表示。
