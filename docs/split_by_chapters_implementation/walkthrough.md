# 指示書 D（章ごと分割機能の実装）実装完了レポート

## 概要

冬之助さんが所有する 42万字クラスの巨大原稿ファイルを、あらかじめ実装された境界検出ロジック（`boundaryDetector`）を活用して章ごとのファイル群に分割する機能（`feature/split-by-chapters` ブランチ）を実装しました。

## 実装内容

### 1. 純粋ロジック層 (`src/utils/splitByChapters.js`)
*   **章境界から分割計画へのパース**: 元ファイルのテキストを読み込み、章やプロローグなどに等分した「セグメントの配列」を `SplitPlan` オブジェクトとして生成します。
*   **安全なファイル名の生成・衝突回避**: Windows / Mac のファイルシステムで無効な文字（`/`, `:`, `<` など）を `_` にサニタイズした上で、元のフォルダに同名ファイル（`1章.txt` など）が既に存在した場合は、自動的に `_2`, `_3` のサフィックスを付与して上書き事故を防ぎます。
*   **変更不可能なステート更新手法**: UI側から「ある章の分割を除外する」「ある章のタイトルを変更する」操作が走った際、元の計画オブジェクトを破壊せず新しい計画オブジェクトを返す不変なアップデート（純粋関数）を徹底しました。

### 2. UI 層と操作フック (`src/components/`, `src/hooks/useSplitByChapters.js`)
*   **操作モーダル**: `ExportPanel`（「編集ツール」セクションの下）に「✂️ 章ごとにファイル分割」ボタンを新設。これをクリックすると画面上にクリーンな確認モーダルが表示されます。
*   **プレビュー機能**: 該当章の部分が、前後400文字の内容（前章の末尾と今章の冒頭）とともにハイライト表示され、どこで文章が切られるかを視覚的に確認できます。
*   **個別操作とタイトル変更**: モーダル上で特定の境界の分割をキャンセルしたり、出力されるファイル名のタイトルをカスタマイズすることが可能です。

### 3. ロバストな I/O と安全性監視 (`useSplitByChapters.js: performSplit`)
*   **安全なパス解決**: `activeFileHandle` がネイティブの Electron IPC 形式 (`{ handle: string }`) か、Web 標準の `FileSystemFileHandle` かを `isNative` フラグで判定し、共通の `fileSystem` アダプター (`fileSystem.createFolder`, `fileSystem.createFile`) を経由させることで、どの環境からでも安全にバックアップフォルダ（`.backup`）や各章ファイルを書き出せるように修正しました。
*   **バックアップ保護**: 分割実行ボタンを押した際、**最初に必ず** `.backup/元ファイル名_original_日時.txt` の完全なコピーを作成します。
*   **アトミック分割**: その後、章ごとに新ファイルをひとつずつ安全に出力します。もしここでエラーが発生した場合はただちに中断し、**生成途中だった新しい章ファイル群を自動的にすべて削除（ロールバック）** して、中途半端な分割状態が残らないよう保護します。

## 手動確認のお願い

AIエージェントの環境上、アプリを立ち上げる目視テストができないため、以下のシナリオの確認を実機にてお願いいたします。

1.  NEXUS を起動し、『■第1章\n（本文）\n■第2章\n（本文）』のような構成を持つダミーのテキストファイルを開いてください。
2.  右側の ExportPanel から「章ごとにファイル分割」ボタンをクリックします。
3.  モーダルが立ち上がり、境界プレビューと章タイトルが表示されることを確認してください。
4.  「分割する」を実行し、該当フォルダに `_01_冒頭.txt`, `_02_第1章.txt` などの複数ファイルが作成されることを確認してください。
5.  同階層に `.backup/` フォルダが作成され、元ファイルが保護退避されていることを確認してください。
