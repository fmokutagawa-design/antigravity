# 実装計画：フェーズ 2 - 対話型・作品隔離型物語整合性エンジン

## 1. 目的
執筆支援アプリ「NEXUS」において、複数の作品（プロジェクト）を並行して執筆していても設定が混ざらず、かつ設定資料（キャラ表等）を「絶対的な正解」として本編の矛盾を自動検知する、高度な物語監査システムを構築する。

## 2. ユーザーレビューが必要な項目
> [!IMPORTANT]
> - **AI による自動提案**: 新しい設定資料らしきファイルを見つけた際、AI が自動で「これは作品 A の設定ですか？」と問い合わせる挙動。
> - **情報の優先順位**: 「設定資料（Master）」>「プロット」>「本編（既刊）」>「草稿」という重み付けの妥当性。
> - **アンドゥ機能**: 知識の関連付けを間違えた際、履歴画面からワンクリックで取り消せる機能の UI。

## 3. オープンクエスチョン
- 作品のルート（境界線）を判定するための特別なファイル（例：`.nexus_project`）を導入すべきか、あるいは単にフォルダ名のみで運用するか。
- 設定資料が「最新ではない」場合（書き換えたが反映し忘れ等）の衝突をどう扱うか。

## 5. 検証計画

### 自動テスト
- 意図的に設定資料と本編で「キャラの瞳の色」や「出身地」を変えたテストデータを用意し、正しくエラーが出るか確認。
- 異なるフォルダ（作品）の知識が混ざっていないことを確認。

### 手動検証
- ユーザーが「設定資料」タグを付けた瞬間に、AI がその事実をデータベースに取り込み、即座に本編の矛盾を指摘し始めるかを確認。
- 関連付けの「取り消し（アンドゥ）」が正しくデータベースに反映されるかを確認。
