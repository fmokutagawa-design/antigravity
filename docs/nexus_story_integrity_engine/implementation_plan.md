# 実装計画：校正監査モード (Story Integrity & Proofreading Engine)

42万字の長編執筆を妨げず、かつ物語の整合性をプロフェッショナルに監査する「校正監査モード」を実装します。

## 1. ユーザーレビューが必要な項目

- **宿題リストの表示形式**: 完全に別ウィンドウにするか、エディタ上に浮遊するフローティングパネルにするか（今回はフローティングパネルを想定）。
- **監査の深さ**: 現在の章のみを対象とする「クイック監査」と、全章を対象とする「フル監査」の選択。

## 2. 提案する変更内容

### A. バックエンド (Python: `proofreader.py`, `bridge_server.py`)
- **[MODIFY] `proofreader.py`**:
    - 設定資料（Materials）をパースし、「生存フラグ」「現在地」「重要アイテムの所持」などを抽出する `StoryStateExtractor` の追加。
    - ユーザーのIF設定を史実よりも優先するロジックの実装。
- **[NEW] `audit_batch_processor.py`**:
    - 42万字をバックグラウンドで分割処理し、進捗を返すバッチエンジン。

### B. フロントエンド (React: `AIAssistant.jsx`, 新規コンポーネント)
- **[NEW] `AuditReportWindow.jsx`**:
    - 独立した「宿題リスト」表示用のフローティングウィンドウ。
    - 指摘事項をクリックすると、エディタの該当行へスクロール＆ハイライトする機能を搭載。
- **[MODIFY] `AIAssistant.jsx`**:
    - 「校正監査モード」ボタンの新設。
    - バッチ処理の進捗バー（%表示）の追加。

## 3. 検証計画

### 自動テスト
- 意図的に「設定資料で死亡しているキャラ」を原稿に登場させ、正しく検出されるか確認。
- 「大阪から稚内へ10分で移動」などの物理的矛盾が検出されるか確認。

### 手動確認
- 42万字相当のダミーテキストでバッチ処理を走らせ、エディタの入力に遅延が発生しないことを確認。
- 宿題リストからのジャンプ機能が正確に動作することを確認。
