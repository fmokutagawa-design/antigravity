# 修正內容の確認 (Walkthrough) - App.jsx の安定化と検索機能の改善

本修正では、検索・置換機能の改善を安全に統合しつつ、リファクタリングの影響で発生していた `App.jsx` のクラッシュ問題を解決しました。

## 実施した変更

### 1. App.jsx のロールバックと構造最適化
- **ロールバック**: `App.jsx` をクラッシュ発生前の安定した状態 (`f3bbf18e`) に一旦戻しました。
- **TDZ (Temporal Dead Zone) 対策**: 製品版ビルドで発生していた「初期化前の変数参照」エラーを防ぐため、`useMaterials` や `handleOpenFile` などの主要な定義をファイル上部に移動し、依存関係の宣言順序を正しく再構成しました。
- **無限ループの防止**: マウント時の初期化処理 (`useEffect`) から `handleOpenFile` への依存を整理し、不要な再レンダリングや無限ループが発生しないように修正しました。
- **機能復旧**: `pendingFileSelect` (別ウィンドウでの自動ファイルオープン) などの、整理中に失われかけていたロジックを確実に復旧しました。

### 2. 本文内検索・置換機能の改善
- **UI の刷新 (`SearchReplace.css`)**: 
    - 背景が透過して文字が重なる問題を修正（確実な不透明背景を設定）。
    - 縦書きモードでもパネルが正しく（横書きで）表示されるよう強制。
    - ダークモードに対応し、視認性を大幅に向上。
- **ロジックの改善 (`SearchReplace.jsx`)**:
    - 入力と同時に検索が走るリアルタイム検索を実装。
    - grep 検索（プロジェクト内検索）の安定性を向上。
    - 置換プレビューモーダルとの連携を強化。

### 3. メモリリークの修正 (`Editor.jsx`)
- **Wheel Listener**: `handleWheel` に `useCallback` を適用し、イベントリスナーが蓄積してパフォーマンスが低下する問題を解消（※ロールバックしたコミットにこの修正が含まれていることを確認済み）。
