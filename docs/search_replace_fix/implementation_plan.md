# 本文内検索・置換機能の改善 実装計画

## 現状の課題
1.  **UIの可読性不足**: `SearchReplace.css` で参照されているCSS変数（`--sidebar-bg` 等）が `index.css` に定義されておらず、背景が透過して本文が透けて見えている。
2.  **検索の即時性欠如**: 検索語入力時に自動で検索が走らず、明示的にボタンを押す必要がある。
3.  **スタイルの不整合**: 縦書きモードのエディタ内でパネルが表示されると、パネル自体も縦書きの影響を受ける可能性がある（スクリーンショットより確認）。

## 修正内容

### 1. UIの修正 (`src/components/SearchReplace.css`)
- 存在するCSS変数 (`--bg-paper`, `--text-main`, `--border-color`, `--accent-color`) への差し替え。
- フォールバック値（ハードコードされた色）の追加。
- パネル全体に `writing-mode: horizontal-tb !important` 等を付与し、縦書きモード下でも正常なUIを維持。
- ダークモード対応の明示的追加。

### 2. 検索ロジックの改善 (`src/components/SearchReplace.jsx`)
- `handleSearch` を `useCallback` でラップし、不必要な再生成を防ぐ。
- `searchTerm` や検索オプションの変更を検知する `useEffect` を追加し、ローカル検索をリアルタイム化。
- インポートに `useCallback` を追加。

## 検証計画
- 検索パネルを表示し、背景が透けていないことを確認。
- 文字を入力した瞬間に「〇〇件見つかりました」等の表示が更新されることを確認。
- 縦書きモードに切り替えた際、検索パネルのレイアウトが崩れないか確認。
- ダークモードでの視認性を確認。
