# プロジェクト横断検索・置換機能の統合とUI改善

長編原稿の全章を対象とした「プロジェクト横断検索・置換」機能を実装し、執筆効率を最大化する。

## 解決した課題
- **嘘検索の解消**: メモリに中身がないファイルでも、検索時にオンデマンドで読み取ることで確実にヒットを拾う。
- **検索範囲の暴走阻止**: OSの Grep が他作品を拾わないよう、検索実行ディレクトリの固定 (cwd) と、サイドバーリストによる厳格なフィルタリングを実装。
- **UIの透明化**: どこを検索しているか、どのエンジン (Grep/JS) を使っているか、どのオプションが有効かを一目でわかるように刷新。

## 提案された変更
### SearchPanel (UI)
- 検索範囲の動的変更機能 (📁 変更ボタン)
- 検索エンジンのステータス表示
- オプション設定の視覚強調 (Aa, .* の点灯)

### Electron (Main Process)
- `fs:grep` ハンドラの修正とパス固定のバグ修正
- 相対パスの絶対パス復元処理の追加

## 検証計画
- 作品全体での「アムロ」検索が正しくヒットすることを確認。
- 他の作品フォルダが検索結果に混ざらないことを確認。
- 検索範囲をサブフォルダに絞り込み、その中だけで結果が出ることを確認。
