# NEXUS 開発・修正 統合レポート (2026-04-27)

最近のセッションで実施した、エディタのパフォーマンス改善、致命的な参照エラーの修正、およびバックエンド・システム層の安定化についてまとめます。

## 1. フロントエンド：エディタ負荷軽減とバグ修正

### ハイライト計算の最適化 [Editor.jsx]
- **問題**: 6万字を超える大規模ファイルで文字を削除・入力した際、CPUが100%に達しフリーズが発生していた。
- **原因**: 強調表示（ハイライト）の計算過程で `substring()` を多用し、マッチごとに巨大な文字列を再生成していた（O(N^2) の負荷）。
- **修正**:
    - `substring()` の生成を廃止し、インデックス指定で直接コードポイントを数える軽量関数 `countCodePoints(str, upTo)` を導入。
    - 大規模テキスト（4万字超）の判定しきい値を同期させ、負荷の高い処理を自動的にスキップする仕組みを強化。

### 参照エラー (ReferenceError) の解消 [App.jsx]
- **問題**: アプリ起動時に `totalManuscriptChars is not defined` となり、画面が真っ白になる現象。
- **原因**: 前回のメモ化修正時に、変数の定義スコープと使用場所（フッター等）に不整合が生じていた。
- **修正**: `totalManuscriptChars` を `footerStats` の `useMemo` 内に統合。スコープを保証し、依存配列を整理することでエラーを完全に解消。

### 再レンダリングの抑制 [useGhostText.js]
- **修正**: ゴーストテキスト（AI提案）が空の場合に `setGhostText` を呼び出さないガードを追加。入力中の不要な `App` 全体の再レンダリングを削減。

## 2. システム層：OneDrive 同期への耐性向上

### ファイル未検出エラーへの対策 [useMaterials.js, ManuscriptPanel.jsx]
- **問題**: OneDrive の同期遅延により、`.nexus/manifest.json` に記載がある章ファイルがローカルに存在しない（雲マーク）場合にエラーが発生。
- **修正**: 
    - ファイル読み込み時に `content` が存在しない場合でも処理をスキップ・継続できるようガードを強化。
    - 「OneDrive を起動」ボタンへの導線を強化し、同期不足によるトラブルをユーザーが解決しやすくした。

### ファイルシステム操作の堅牢化 [fileSystem.electron.js]
- バイナリ書き込みやディレクトリ操作のエラーハンドリングを強化し、macOS/Windows 間での挙動差を軽減。
