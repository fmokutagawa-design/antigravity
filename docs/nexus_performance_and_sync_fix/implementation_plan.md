# 実装計画：エディタ負荷軽減とシステム安定化

## 概要
6万字超のドキュメントにおける入力・消去時の極端な遅延の解消、およびプログラム停止エラーの修正、OneDrive 環境下での動作安定化を目的とする。

## 提案される変更

### 1. エディタパフォーマンスの根本改善
- **ハイライト計算の軽量化**: `regex.exec` のループ内で `substring()` を生成していた箇所を、インデックス指定でカウントするロジックに変更し、メモリ確保と GC 負荷を最小化する。
- **しきい値の調整**: 座標キャッシュやハイライト描画をスキップする `isMassiveText` の判定を、ハイライト計算ロジック側にも適用する。

### 2. 致命的エラーの修正
- **スコープ不整合の解消**: `totalManuscriptChars` が未定義となるバグを、計算ロジックを既存の安定した `useMemo`（footerStats）に統合することで修正する。

### 3. OneDrive 連携の安定化
- **未同期ファイルのハンドリング**: `useMaterials` フックにおいて、実体がない（ダウンロードされていない）ファイルを読み込もうとした際の例外処理を強化し、アプリが停止しないようにする。
