# 真の統合版ビルド計画 (TRUE_MERGE_PLAN)

## 概要
私の誤った「リーダーモード復元作業」により、その直近に実装されていた「画像ドラッグ＆ドロップ」機能や「パフォーマンスパッチ」の一部が消失（デグレ）してしまいました。これらすべての最新機能を「共存」させた状態で完全復旧させます。

## ユーザーレビューが必要な項目
> [!IMPORTANT]
> **画像ドラッグ＆ドロップの復活**: 履歴から `handleImageDrop` などのロジックを抽出し、再実装します。
> **パフォーマンスパッチの完全適用**: `Editor.jsx` 側の修正（ハイライト制限など）を再適用します。
> **リーダーモードとの共存**: プレビュー（原稿用紙等）を破壊せず、リーダーモードを独立した機能として配置します。

## 変更内容の詳細

### 1. src/App.jsx [MODIFY]
- **状態定義の順序整理**: `ReferenceError` を防ぐため、`useGhostText` などのフックよりも前に必要な `useState` をすべて定義します。
- **画像D&D復元**: `handleImageDrop` 関数を、コミット `1d0e5f1` から抽出して再実装します。
- **リーダーモード統合**:
  - `ReaderView` コンポーネントをインポート。
  - `showReader` 状態を定義。
  - `Alt+R` ショートカットを実装。
  - フッターに「📖 リーダー」ボタンを追加（「プレビュー」ボタンはそのまま）。
  - `ReaderView` をオーバーレイ表示。

### 2. src/components/Editor.jsx [MODIFY]
- **画像D&D復元**: `onImageDrop` prop を受け取り、`handleDrop`, `handleDragOver` ハンドラを実装して画像を `App.jsx` に渡すようにします。
- **パフォーマンスパッチ再適用**:
  - `MAX_HIGHLIGHT_ELEMENTS = 2000` を定義し、描画負荷を制限。
  - コンポーネントを `React.memo` でラップし、不要な再レンダリングを抑止。
