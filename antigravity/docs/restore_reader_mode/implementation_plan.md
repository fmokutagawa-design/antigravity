# リーダーモードの復元実装計画

ユーザーより「リーダーモードが消えた」との指摘を受け、前回の `App.jsx` 大規模リファクタリング時に誤って変更または削除された可能性のあるUIラベルおよび状態管理を特定し、復元します。

## ユーザーレビューが必要な項目
- **UIラベルの名称**: 現在「プレビュー」となっているボタンを「リーダー」に戻します。
- **挙動の確認**: 「リーダー」に切り替えた際、以前はサイドバーが自動で閉じる等の挙動があったか、ユーザーの期待に沿う形に調整します。

## 変更内容

### [NEXUS App]

#### [MODIFY] [App.jsx](file:///Volumes/Black6T/Nexus_Dev/antigravity/src/App.jsx)
- `activeTab` の初期値コメントを修正 (`// 'editor', 'preview', 'reader' ...`)。
- フッターの切り替えボタンのテキストを「プレビュー」から「リーダー」に変更。
- `activeTab === 'preview'` を `activeTab === 'reader'` (または同等) に名称変更することで、ユーザーの呼び方に合わせる。
- `paperStyle === 'clean'` 選択時、または「リーダー」タブ選択時に、没入感を高めるためのUI調整（サイドバーの自動非表示など）が以前あったか再確認し、必要なら実装。

#### [MODIFY] [Toolbar.jsx](file:///Volumes/Black6T/Nexus_Dev/antigravity/src/components/Toolbar.jsx)
- 設定パネル内の「プレビュー」関連のテキストを「リーダー」に合わせて調整。

## 修正の根拠
- `activeTab` の切り替えボタンにおいて、以前のバージョンでは「リーダー」という呼称が使われていた可能性が高い。
- `paperStyle: 'clean'` は「リーダーモード」として親しまれているため、UI上の整合性を取る。

## 検証計画
### 手動確認
- フッターの「リーダー」ボタンをクリックし、正常に閲覧画面（プレビュー画面）に切り替わること。
- サイドバーの表示/非表示を切り替え、「リーダーモード」としての使い勝手を確認。
- `paperStyle` を「クリーン」にした際の挙動が以前と同様であることを確認。
