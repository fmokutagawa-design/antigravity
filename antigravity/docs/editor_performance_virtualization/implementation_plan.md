# エディタ極限高速化計画：座標計算と描画の仮想化（Virtualization）

## 1. 目的
大規模な長編原稿（14万字〜40万字）を執筆する際、JavaScript の全文走査・座標計算によるメインスレッドの占有（フリーズ）を解消し、常に 0ms に近い快適な入力速度と、視認性を高めるハイライト（着色）機能を両立させます。

## 2. ユーザーレビューが必要な事項
> [!IMPORTANT]
> ### 仮想化の影響について
> - **計算範囲の限定**: 14万文字すべてではなく「現在見えている数千文字」だけを計算対象にします。
> - **高速スクロール時の挙動**: 超高速でスクロールした際、ハイライトの着色が 0.1秒ほど遅れて追いつく場合があります。これは「入力の軽さ」を最優先するための意図的な挙動です。
>
> ### 安全性の保証
> - 本修正は「背面の装飾レイヤー」の計算方式のみを変更します。
> - **保存処理や文章の実体（textarea）の管理ロジックには一切触れない**ため、ファイルが消失したりコピペができなくなったりするリスクは理論上排除されています。

## 3. 提案される変更点

### Editor.jsx の刷新

#### [MODIFY] [Editor.jsx](file:///Volumes/Black6T/Nexus_Dev/antigravity/src/components/Editor.jsx)
- **段落インデックス（Paragraph Map）の導入**:
  - `localTextRef.current` を `\n` で分割し、各段落の「開始行数」と「文字オフセット」のみを保持する軽量な Map を作成します（数百件程度なので一瞬で終わります）。
- **`charPositionsCache` の仮想化**:
  - 全文の Map 作成（14万件）を廃止します。
  - 現在の `scrollTop` から「今見えている段落」を特定し、その範囲（前後数ページ分のバッファを含む）だけを文字単位で座標計算します。
- **ハイライト制限（20万字制限）の撤廃**:
  - 仮想化により負荷が一定（O(1)）になるため、どんな長編でも着色が表示されるように戻します。
- **パフォーマンス・モニター（LAG モニター）の実装**:
  - 画面隅に「JavaScript の計算時間」をリアルタイム表示し、改善効果を数値で証明します。

## 4. 懸念されるリスクと回避策

| リスク | 対策 |
| :--- | :--- |
| **ハイライトのズレ** | CSS の `line-height` 等と計算ロジックを一貫させ、座標計算を段落単位で独立させることで累積誤差を防ぎます。 |
| **ジャンプ時の未着色** | 検索や章ジャンプで `applyText` が呼ばれた際、ジャンプ先の計算を「最優先・即時」で実行する割り込み処理を入れます。 |
| **スクロールのチラつき** | 表示範囲の上下に「バッファ領域（数千文字分）」を確保し、通常のスクロールでは計算待ちが発生しないようにします。 |

## 5. 検証プラン

### 自動・ブラウザテスト
- 40万字のファイルを読み込み、入力から反映までの JS ラグが 常時 16ms（60fps）以内であることをモニターで確認。
- 縦書き・横書きの両方で、スクロール追従性と着色の正確性を確認。

### 手動確認
- ユーザー様に、これまで「死ぬほど重かった」ファイルでの書き味が劇的に改善していることを確認いただく。
- コピペ、範囲選択、UNDO/REDO が従来通り安定して動作することを確認。
