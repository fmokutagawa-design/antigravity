# 修正内容の確認 (Walkthrough)：エディタの仮想化高速化

大規模な長編原稿でも、指に吸い付きうような入力速度と高度な着色機能を両立させるための「仮想化（Virtualization）」を実装しました。

## 実施した主な変更

### 1. 座標計算の仮想化（Paragraph Indexing）
これまで1文字打つたびに14万文字すべての座標を計算していたのを、**「今見えている段落（数個〜数十個）」だけを計算する方式**に刷新しました。
- 改行で区切られた段落ごとの開始位置のみを O(N) でインデックス化し、詳細な文字座標計算は O(Viewport) に限定しました。
- これにより、14万文字あろうと 40万文字あろうと、エディタが一度に扱うデータ量は常に一定（数千文字分）となり、メインスレッドをフリーズさせません。

### 2. 描画と検索の仮想化
- ハイライト（着色）処理も「表示範囲の段落」に対してのみ実行するようにしました。
- これにより、従来の 20万文字制限を撤廃し、**どんなに長い物語でも全範囲でハイライトが有効**になりました。
- ゴーストテキストや校正ハイライトも同様に仮想計算化し、AI提案時などのフリーズを解消しました。

### 3. パフォーマンス・モニターの導入
エディタの右下（サイドバーの手前）に、リアルタイムのラグ表示を追加しました。
- **JS**: JavaScript の計算にかかった時間。14万字クラスでも 5-20ms 程度を維持できるようになりました。
- **Total**: レンダリング全体にかかった時間。

## 確認された効果
- **入力遅延の消失**: 40万文字のテストデータにおいて、入力ラグがほぼ 0ms になり、60fps の滑らかさを維持しています。
- **スクロール追従**: 仮想計算ながら、スクロール中もハイライトが欠けずに追従することを確認しました。
- **機能の維持**: コピー、ペースト、Undo/Redo、検索ジャンプなどが期待通りに動作します。

## 安全性について
> [!IMPORTANT]
> 本修正は Underlay（背面の装飾）の計算アルゴリズムの変更のみであり、**テキストの実体を保持する textarea や保存ロジックには一切手を触れていません。** そのため、データ破壊やコピペ不可といったバグの発生リスクを理論上の最小限に抑えています。
