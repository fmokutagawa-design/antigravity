# NEXUS エンジン修復・完成 実施計画

## 目的
Opus による診断結果に基づき、NEXUS 校正・監査エンジンの不具合修正、精度向上、およびプロジェクト隔離を完遂する。

## 実施内容

### 1. Python バックエンドの修復
- `bridge_server.py` に不足していた UI 連携用エンドポイントを全て実装する。
- `ingest_novels.py` の同期エラーを解消し、確実にデータベースが更新されるようにする。

### 2. 監査ロジックの高度化
- `Proofreader.py` において、静的ルールと物語監査（属性照合）を分離・統合する。
- 瞳の色、出身、武器などの矛盾を正規表現とコンテキストマッチングで検知する。

### 3. パフォーマンスと精度の最適化
- 誤検知の多い短いパターンを整理し、校正のノイズを減らす。
- 作品ごとの `project_id` をパスから自動判別し、RAG 検索範囲を限定する。

### 4. ユーザー体験の向上
- 長時間のバッチ監査を非同期化し、UI に進捗を表示する。
- 宿題リストにプロジェクト情報を追加し、複数作品管理を容易にする。
